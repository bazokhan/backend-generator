# Example 02: Blog

Multiple related models, JWT auth, React Admin dashboard generated by `tgraph all`.

## What This Demonstrates

- Multiple models with relations (`Post → User`, `Post → Category`)
- `// @tg_label(name)` for relation display fields
- `/// @tg_upload(image)` for file upload fields
- `/// @tg_format(email/password)` for field formatting
- JWT authentication with `JwtAuthGuard`
- React Admin dashboard with `authProvider`

## Setup

```bash
npm install
```

For the dashboard:

```bash
cd src/dashboard && npm install && cd ../..
```

## Step 1: Database Migration

```bash
npx prisma migrate dev --name init
```

## Step 2: Generate Static Files

```bash
npx tgraph static --include paginated-search-query.dto,paginated-search-result.dto,api-response.dto,pagination.interceptor,paginated-search.decorator,paginated-search.util
```

## Step 3: Generate API + Dashboard

```bash
npx tgraph all
```

This generates for each model (User, Post, Category):
- Controller with `@UseGuards(JwtAuthGuard)`
- Service with CRUD + unique field getters
- Create/Update DTOs with validation
- React Admin List/Edit/Create/Show/Studio pages
- Updates `App.tsx` with resource registrations

## Step 4: Start the API

```bash
npm run start:dev
```

API at http://localhost:3000, Swagger at http://localhost:3000/api

## Step 5: Start the Dashboard

```bash
cd src/dashboard && npm run dev
```

Dashboard at http://localhost:5173

## Seed Data (Optional)

Create a user via the API to log into the dashboard:

```bash
curl -X POST http://localhost:3000/tg-api/users \
  -H "Content-Type: application/json" \
  -d '{"name": "Admin", "email": "admin@example.com", "password": "password123"}'
```

Then log in at the dashboard with those credentials.

```
│   │   ├── user.admin.tg.service.ts

```
