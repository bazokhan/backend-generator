# Example 02: Blog

Multiple related models, JWT auth, React Admin dashboard generated by `tgraph all`.

## What This Demonstrates

- Multiple models with relations (`Post → User`, `Post → Category`)
- `// @tg_label(name)` for relation display fields
- `/// @tg_upload(image)` for file upload fields
- `/// @tg_format(email/password)` for field formatting
- JWT authentication with `JwtAuthGuard`
- React Admin dashboard with `authProvider`

## Quick Start

```bash
npm install
cp .env.example .env
npx prisma migrate dev --name init
npx tgraph all
npm run seed
npm run build && npm run start
```

API at **http://localhost:3002**, Swagger at **http://localhost:3002/api**

## Step by Step

### 1. Install dependencies

```bash
npm install
```

For the dashboard (after running `tgraph all`):

```bash
cd src/dashboard && npm install && cd ../..
```

### 2. Set up the database

```bash
cp .env.example .env
npx prisma migrate dev --name init
```

### 3. Generate API + Dashboard

```bash
npx tgraph all
```

This generates for each model (User, Post, Category):
- Controller with `@UseGuards(JwtAuthGuard)`
- Service with CRUD + unique field getters
- Create/Update DTOs with validation
- React Admin List/Edit/Create/Show/Studio pages
- Updates `App.tsx` with resource registrations
- Scaffolds `src/dashboard/` Vite project

### 4. Seed the database

Creates an admin user for dashboard login:

```bash
npm run seed
```

Seed credentials:
- **Email:** admin@example.com
- **Password:** admin123

### 5. Start the API

```bash
npm run start:dev
```

API at http://localhost:3002, Swagger at http://localhost:3002/api

### 6. Start the Dashboard

```bash
cd src/dashboard && npm run dev
```

Dashboard at http://localhost:5173 — log in with `admin@example.com` / `admin123`

## Architecture

```
src/
├── auth/                          ← hand-written JWT auth
│   ├── auth.controller.ts         ← POST /auth/login
│   ├── auth.service.ts
│   ├── auth.module.ts
│   ├── jwt.strategy.ts
│   └── jwt-auth.guard.ts
├── infrastructure/database/
│   └── prisma.service.ts
├── user/                          ← generated by tgraph
├── post/                          ← generated by tgraph
├── category/                      ← generated by tgraph
├── dtos/                          ← generated by tgraph
├── decorators/                    ← generated by tgraph
├── interceptors/                  ← generated by tgraph
└── dashboard/src/
    ├── App.tsx                    ← scaffolded + updated by tgraph
    ├── providers/
    │   ├── authProvider.ts        ← scaffolded by tgraph
    │   └── fieldDirectives.generated.ts
    └── resources/                 ← generated by tgraph
        ├── users/
        ├── posts/
        └── categories/
```
