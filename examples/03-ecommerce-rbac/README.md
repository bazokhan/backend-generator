# Example 03: E-commerce RBAC

Full showcase. Five models, role-based access control (ADMIN/CUSTOMER), file uploads, enums, relations, and a React Admin dashboard. Demonstrates the full power of TGraph.

## What This Demonstrates

- 5 models with complex relations (User → Order → OrderItem → Product → Category)
- `enum Role { ADMIN, CUSTOMER }` — RBAC with `RolesGuard`
- `enum OrderStatus` — enum field support
- `/// @tg_upload(image)` — file upload fields
- `adminGuards` — separate auth guard from authorization guard
- React Admin dashboard requiring admin login
- Seed data script for quick setup

## Setup

```bash
npm install
```

For the dashboard:

```bash
cd src/dashboard && npm install && cd ../..
```

## Step 1: Database Migration

```bash
npx prisma migrate dev --name init
```

## Step 2: Generate Static Files

```bash
npx tgraph static --include paginated-search-query.dto,paginated-search-result.dto,api-response.dto,pagination.interceptor,paginated-search.decorator,paginated-search.util
```

## Step 3: Generate API + Dashboard

```bash
npx tgraph all
```

This generates for all 5 models (User, Category, Product, Order, OrderItem):
- Controller with `@UseGuards(JwtAuthGuard, RolesGuard)` — admin-only
- Service with CRUD
- Create/Update DTOs with validation
- React Admin pages with relation support

## Step 4: Seed the Database

```bash
npm run seed
```

Creates:
- Admin user: `admin@example.com` / `admin123`
- 2 categories (Electronics, Clothing)
- 3 products

## Step 5: Start the API

```bash
npm run start:dev
```

API at `http://localhost:3000`, Swagger at `http://localhost:3000/api`

## Step 6: Start the Dashboard

```bash
cd src/dashboard && npm run dev
```

Dashboard at `http://localhost:5173`

## Architecture

```
src/
├── auth/
│   ├── jwt.strategy.ts        ← reads role from JWT payload
│   ├── jwt-auth.guard.ts
│   ├── roles.guard.ts         ← checks role === 'ADMIN'
│   ├── auth.module.ts
│   ├── auth.service.ts
│   └── auth.controller.ts     ← POST /auth/login (returns role in token)
├── infrastructure/database/
│   └── prisma.service.ts
├── features/                  ← generated by tgraph
│   ├── user/
│   ├── category/
│   ├── product/
│   ├── order/
│   └── order-item/
└── dashboard/src/
    ├── App.tsx                ← updated by tgraph
    ├── providers/
    │   └── authProvider.ts   ← checks role on login
    └── resources/             ← generated by tgraph
```

## Key Config: `adminGuards`

The `tgraph.config.ts` uses `adminGuards` to apply `RolesGuard` only when `requireAdmin: true`:

```typescript
export const config: UserConfig = {
  authenticationEnabled: true,
  requireAdmin: true,
  guards: [{ name: 'JwtAuthGuard', importPath: '@/auth/jwt-auth.guard' }],
  adminGuards: [{ name: 'RolesGuard', importPath: '@/auth/roles.guard' }],
};
```

Generated controllers:

```typescript
@Controller('admin-api/users')
@UseGuards(JwtAuthGuard, RolesGuard)  // ← both guards applied
export class UserAdminController { ... }
```
