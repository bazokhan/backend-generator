# Example 03: E-commerce RBAC

Full showcase. Five models, role-based access control (ADMIN/CUSTOMER), file uploads, enums, relations, and a React Admin dashboard. Demonstrates the full power of TGraph.

## What This Demonstrates

- 5 models with complex relations (User ‚Üí Order ‚Üí OrderItem ‚Üí Product ‚Üí Category)
- `enum Role { ADMIN, CUSTOMER }` ‚Äî RBAC with `RolesGuard`
- `enum OrderStatus` ‚Äî enum field support
- `/// @tg_upload(image)` ‚Äî file upload fields
- `adminGuards` ‚Äî separate auth guard from authorization guard
- React Admin dashboard requiring admin login
- Seed data script for quick setup

## Quick Start

```bash
npm install
cp .env.example .env
npx prisma migrate dev --name init
npx tgraph all
npm run seed
npm run build && npm run start
```

API at **http://localhost:3003**, Swagger at **http://localhost:3003/api**

## Step by Step

### 1. Install dependencies

```bash
npm install
```

For the dashboard (after running `tgraph all`):

```bash
cd src/dashboard && npm install && cd ../..
```

### 2. Set up the database

```bash
cp .env.example .env
npx prisma migrate dev --name init
```

### 3. Generate API + Dashboard

```bash
npx tgraph all
```

This generates for all 5 models (User, Category, Product, Order, OrderItem):
- Controller with `@UseGuards(JwtAuthGuard, RolesGuard)` ‚Äî admin-only
- Service with CRUD
- Create/Update DTOs with validation
- React Admin pages with relation support

### 4. Seed the Database

```bash
npm run seed
```

Creates:
- Admin user: `admin@example.com` / `admin123`
- 2 categories (Electronics, Clothing)
- 3 products

## Step 5: Start the API

```bash
npm run start:dev
```

API at `http://localhost:3003`, Swagger at `http://localhost:3003/api`

## Step 6: Start the Dashboard

```bash
cd src/dashboard && npm run dev
```

Dashboard at `http://localhost:5173`

## Key Config: `adminGuards`

The `tgraph.config.ts` uses `adminGuards` to apply `RolesGuard` only when `requireAdmin: true`:

```typescript
export const config: UserConfig = {
  authenticationEnabled: true,
  requireAdmin: true,
  guards: [{ name: 'JwtAuthGuard', importPath: '@/auth/jwt-auth.guard' }],
  adminGuards: [{ name: 'RolesGuard', importPath: '@/auth/roles.guard' }],
};
```

Generated controllers:

```typescript
@Controller('admin-api/users')
@UseGuards(JwtAuthGuard, RolesGuard)  // ‚Üê both guards applied
export class UserAdminController { ... }
```
