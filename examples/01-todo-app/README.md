# Example 01: Todo App

The simplest possible TGraph example. No auth, no dashboard — just a clean REST API with pagination.

## What This Demonstrates

- `// @tg_form()` marks a model for generation
- `/// @tg_readonly` fields excluded from create/update DTOs
- Flat `UserConfig` with auth disabled
- `nonInteractive: true` for scripted generation

## Quick Start

```bash
npm install
cp .env.example .env
npx prisma migrate dev --name init
npx tgraph all
npm run build
npm run start
```

API at **http://localhost:3001**, Swagger at **http://localhost:3001/api**

## Step by Step

### 1. Install dependencies

```bash
npm install
```

### 2. Set up the database

```bash
cp .env.example .env
npx prisma migrate dev --name init
```

### 3. Generate the API

```bash
npx tgraph all
```

This generates:
- `src/todo/todo.controller.ts` — REST controller with `GET /api/todos`, `POST /api/todos`, etc.
- `src/todo/todo.service.ts` — CRUD service with Prisma
- `src/todo/create-todo.dto.ts` — DTO (title, description, completed)
- `src/todo/update-todo.dto.ts` — partial update DTO
- `src/dtos/` — pagination and response DTOs
- `src/app.module.ts` — updated with TodoModule import

### 4. Start the server

```bash
npm run start:dev
```

### 5. Try it

Open Swagger UI at **http://localhost:3001/api**

Or use curl:

```bash
# Create a todo
curl -X POST http://localhost:3001/api/todos \
  -H "Content-Type: application/json" \
  -d '{"title": "Buy groceries", "description": "Milk, eggs, bread"}'

# List todos (paginated)
curl "http://localhost:3001/api/todos?page=1&limit=10"

# Mark complete
curl -X PUT http://localhost:3001/api/todos/<id> \
  -H "Content-Type: application/json" \
  -d '{"completed": true}'

# Delete
curl -X DELETE http://localhost:3001/api/todos/<id>
```
