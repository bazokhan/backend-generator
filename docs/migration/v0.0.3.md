# Migration Guide: v0.0.2 → v0.0.3

This guide helps you migrate from the old flat configuration structure to the new organized configuration system in v0.0.3.

## Breaking Changes

### 1. Configuration Structure

The configuration object has been completely restructured into logical sections.

### 2. CLI Flags Removed

Most CLI flags for overriding config values have been removed. Use multiple config files instead.

### 3. Module Path Resolution

Module path resolution now uses fully configurable search paths. You can organize your modules in any folder structure you prefer.

---

## Configuration Migration

### Old Config (v0.0.2)

```typescript
export const config: Config = {
  schemaPath: 'prisma/schema.prisma',
  dashboardPath: 'src/dashboard/src',
  dtosPath: 'src/dtos/generated',
  suffix: 'Tg',
  isAdmin: true,
  updateDataProvider: true,
  nonInteractive: false,
  paths: {
    appModule: 'src/app.module.ts',
    moduleRoots: {
      features: ['src/features'],
      infrastructure: ['src/infrastructure'],
    },
    dashboard: {
      appComponent: 'src/dashboard/src/App.tsx',
      dataProvider: 'src/dashboard/src/providers/dataProvider.ts',
    },
  },
};
```

### New Config (v0.0.3)

```typescript
export const config: Config = {
  input: {
    schemaPath: 'prisma/schema.prisma',
    prismaService: 'src/infrastructure/database/prisma.service.ts',
  },

  output: {
    backend: {
      dtos: 'src/dtos/generated',
      modules: {
        searchPaths: ['src/features', 'src/infrastructure', 'src'],
        defaultRoot: 'src/features',
      },
      staticFiles: {
        guards: 'src/guards',
        decorators: 'src/decorators',
        dtos: 'src/dtos',
        interceptors: 'src/interceptors',
        utils: 'src/utils',
      },
    },
    dashboard: {
      root: 'src/dashboard/src',
      resources: 'src/dashboard/src/resources',
    },
  },

  api: {
    suffix: 'Tg',
    prefix: 'tg-api',
    authentication: {
      enabled: true,
      requireAdmin: true,
      guards: [
        { name: 'JwtAuthGuard', importPath: '@/guards/jwt-auth.guard' },
        { name: 'AdminGuard', importPath: '@/guards/admin.guard' },
      ],
    },
  },

  dashboard: {
    enabled: true,
    updateDataProvider: true,
    components: {
      form: {},
      display: {},
    },
  },

  behavior: {
    nonInteractive: false,
  },

  paths: {
    appModule: 'src/app.module.ts',
    dataProvider: 'src/dashboard/src/providers/dataProvider.ts',
    appComponent: 'src/dashboard/src/App.tsx',
  },
};
```

## Field Mapping Table

| Old Field                          | New Field                            | Notes                           |
| ---------------------------------- | ------------------------------------ | ------------------------------- |
| `schemaPath`                       | `input.schemaPath`                   | Moved to input section          |
| `dashboardPath`                    | `output.dashboard.root`              | Moved to output section         |
| `dtosPath`                         | `output.backend.dtos`                | Moved to output section         |
| `suffix`                           | `api.suffix`                         | Moved to API section            |
| `isAdmin`                          | `api.authentication.requireAdmin`    | Renamed and moved               |
| `updateDataProvider`               | `dashboard.updateDataProvider`       | Moved to dashboard section      |
| `nonInteractive`                   | `behavior.nonInteractive`            | Moved to behavior section       |
| `paths.appModule`                  | `paths.appModule`                    | Kept in paths (optional)        |
| `paths.moduleRoots.features`       | `output.backend.modules.searchPaths` | Combined into searchPaths array |
| `paths.moduleRoots.infrastructure` | `output.backend.modules.searchPaths` | Combined into searchPaths array |
| `paths.dashboard.appComponent`     | `paths.appComponent`                 | Simplified                      |
| `paths.dashboard.dataProvider`     | `paths.dataProvider`                 | Simplified                      |

## Migration Steps

### Step 1: Backup Current Config

```bash
cp tgraph.config.ts tgraph.config.ts.backup
```

### Step 2: Update Config File

Option A: **Manual Update** - Follow the field mapping table above

Option B: **Regenerate** - Delete and recreate config:

```bash
rm tgraph.config.ts
tgraph init
# Then customize the generated config
```

### Step 3: Test Configuration

Run diagnostics to verify:

```bash
tgraph doctor
```

### Step 4: Regenerate Code

```bash
tgraph all
```

### Scenario 3: Multiple APIs

Instead of using CLI flags, create separate config files:

**tgraph.admin.config.ts:**

```typescript
export const config: Config = {
  input: {
    schemaPath: 'prisma/schema.prisma',
    prismaService: 'src/infrastructure/database/prisma.service.ts',
  },
  output: {
    backend: {
      dtos: 'src/dtos/admin',
      // ... other config
    },
  },
  api: {
    suffix: 'Admin',
    prefix: 'admin-api',
    authentication: {
      enabled: true,
      requireAdmin: true,
      guards: [
        { name: 'JwtAuthGuard', importPath: '@/guards/jwt-auth.guard' },
        { name: 'AdminGuard', importPath: '@/guards/admin.guard' },
      ],
    },
  },
  // ... other config
};
```

**tgraph.public.config.ts:**

```typescript
export const config: Config = {
  input: {
    schemaPath: 'prisma/schema.prisma',
    prismaService: 'src/infrastructure/database/prisma.service.ts',
  },
  output: {
    backend: {
      dtos: 'src/dtos/public',
      // ... other config
    },
  },
  api: {
    suffix: 'Public',
    prefix: 'api',
    authentication: {
      enabled: true,
      requireAdmin: false,
      guards: [{ name: 'JwtAuthGuard', importPath: '@/guards/jwt-auth.guard' }],
    },
  },
  // ... other config
};
```

Generate both:

```bash
tgraph api --config tgraph.admin.config.ts
tgraph api --config tgraph.public.config.ts
```

## CLI Changes

### Removed Flags

The following CLI flags have been removed. Use config files instead:

| Removed Flag                | New Approach                                           |
| --------------------------- | ------------------------------------------------------ |
| `--schema <path>`           | Set `input.schemaPath` in config                       |
| `--dashboard <path>`        | Set `output.dashboard.root` in config                  |
| `--dtos <path>`             | Set `output.backend.dtos` in config                    |
| `--suffix <name>`           | Set `api.suffix` in config                             |
| `--admin`                   | Set `api.authentication.requireAdmin: true` in config  |
| `--no-admin`                | Set `api.authentication.requireAdmin: false` in config |
| `--update-data-provider`    | Set `dashboard.updateDataProvider: true` in config     |
| `--no-update-data-provider` | Set `dashboard.updateDataProvider: false` in config    |

### New Flags

| Flag                             | Description                |
| -------------------------------- | -------------------------- |
| `--config <path>` or `-c <path>` | Specify custom config file |
| `--file <path>`                  | Alias for --config         |

### Kept Flags

| Flag                           | Description               |
| ------------------------------ | ------------------------- |
| `-y, --yes, --non-interactive` | Auto-confirm prompts      |
| `--interactive`                | Force interactive prompts |
| `-h, --help`                   | Show help                 |

## New Features in v0.0.3

### 1. Multiple Config Files

Generate different APIs with separate configs:

```bash
tgraph api --config tgraph.admin.config.ts
tgraph api --config tgraph.public.config.ts
```

### 2. Component Overrides

Customize React Admin components:

```typescript
dashboard: {
  components: {
    form: {
      TextInput: {
        name: 'CustomTextInput',
        importPath: '@/components/custom/TextInput'
      },
    },
  },
}
```

### 3. Configurable Guards

Configure authentication guards per API:

```typescript
api: {
  authentication: {
    guards: [
      { name: 'JwtAuthGuard', importPath: '@/guards/jwt-auth.guard' },
      { name: 'AdminGuard', importPath: '@/guards/admin.guard' },
      { name: 'RateLimitGuard', importPath: '@/guards/rate-limit.guard' },
    ],
  },
}
```

### 4. Flexible Module Search

Configure where to search for modules:

```typescript
output: {
  backend: {
    modules: {
      searchPaths: ['src/features', 'src/domains', 'src/modules', 'src'],
      defaultRoot: 'src/features',
    },
  },
}
```

### 5. Static File Locations

Control where static files are generated:

```typescript
output: {
  backend: {
    staticFiles: {
      guards: 'src/common/guards',
      decorators: 'src/common/decorators',
      dtos: 'src/common/dtos',
      interceptors: 'src/common/interceptors',
    },
  },
}
```

## Troubleshooting

### Issue: Config validation errors

**Error:** `Config file is missing required section 'input'`

**Solution:** Ensure all required sections exist:

- `input`
- `output`
- `api`
- `dashboard`
- `behavior`

### Issue: Module not found

**Error:** `Module 'User' not found`

**Solution:** Update `output.backend.modules.searchPaths` to include your module directories.

### Issue: Guards not applied

**Error:** Guards not showing up in generated controllers

**Solution:** Check `api.authentication.guards` configuration and ensure guards exist at specified paths.

### Issue: Components not overridden

**Error:** Custom components not being used

**Solution:**

1. Verify component paths in `dashboard.components`
2. Ensure components are properly exported
3. Regenerate dashboard: `tgraph dashboard`
